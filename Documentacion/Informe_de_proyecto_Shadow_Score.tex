\section{Introducción}

El proyecto \textbf{Shadow-Score Académico} surge como una herramienta tecnológica para visibilizar y cuantificar el impacto de la carga doméstica no remunerada en el rendimiento académico de estudiantes universitarios, con un enfoque diferencial por género y estrato socioeconómico. Enmarcado en el \textbf{ODS 5 (Igualdad de Género)} y particularmente en la meta \textbf{5.4} (reconocimiento y redistribución del trabajo doméstico y de cuidados no remunerado), el proyecto busca evidenciar la ``pobreza de tiempo'' que afecta desproporcionadamente a las mujeres de estratos bajos en Colombia.

La solución consiste en un \textbf{simulador interactivo} que, a partir de datos personales y de uso del tiempo, calcula un índice de penalización (Shadow-Score) y proyecta escenarios de corresponsabilidad mediante la comparación de ``gemelos digitales''. Este informe detalla la propuesta técnica, el flujo de trabajo y el plan de desarrollo para construir un producto robusto y escalable, utilizando \textbf{Streamlit} como plataforma principal.

\section{Objetivos}

\subsection{Objetivo general}
Desarrollar una aplicación web interactiva que permita a estudiantes universitarios estimar el efecto de la carga doméstica en su rendimiento académico y visualizar el potencial beneficio de una distribución equitativa de las tareas del hogar.

\subsection{Objetivos específicos}
\begin{itemize}
    \item Implementar un modelo matemático de penalización basado en fatiga cognitiva y costo de oportunidad.
    \item Diseñar una interfaz de usuario intuitiva para la captura de datos (género, estrato, horas domésticas, horas de estudio, promedio actual).
    \item Calcular en tiempo real el Shadow-Score, la fatiga cognitiva, las horas de estudio efectivas y el promedio estimado.
    \item Simular un escenario de corresponsabilidad (reducción de carga doméstica) y mostrar la comparación con la situación actual.
    \item Visualizar los resultados mediante gráficos claros (barras, medidores, batería cognitiva) que faciliten la comprensión del fenómeno.
    \item Permitir la exportación de resultados (opcional) y, en futuras versiones, almacenar datos anónimos para investigación.
\end{itemize}

\section{Descripción de la solución}

La aplicación se construirá íntegramente en Python utilizando \textbf{Streamlit}, un framework que convierte scripts en aplicaciones web interactivas sin necesidad de conocimientos de frontend. La lógica de negocio residirá en módulos separados, garantizando mantenibilidad y posibilidad de reutilización (por ejemplo, para una futura API).

\subsection{Componentes principales}

\begin{enumerate}
    \item \textbf{Módulo de modelo matemático} (\texttt{modelo.py}):
    \begin{itemize}
        \item Funciones para calcular carga total, fatiga cognitiva (con ajustes por género y estrato), horas de estudio efectivas, promedio estimado y Shadow-Score.
        \item Parámetros basados en estudios sociológicos colombianos (encuestas de uso del tiempo del DANE, investigaciones sobre rendimiento académico).
    \end{itemize}
    \item \textbf{Módulo de utilidades} (\texttt{utils.py}):
    \begin{itemize}
        \item Validaciones de entrada (ej. suma de horas \(\leq 168\)).
        \item Formateo de números y mensajes.
        \item Funciones de ayuda para gráficos.
    \end{itemize}
    \item \textbf{Aplicación principal} (\texttt{app.py}):
    \begin{itemize}
        \item Interfaz de usuario con formularios, botones y áreas de resultados.
        \item Uso de \texttt{st.session\_state} para mantener el estado entre interacciones (actual vs. simulado).
        \item Visualizaciones con \texttt{st.metric}, \texttt{st.progress}, \texttt{st.plotly\_chart} (gráficos de Plotly) y componentes personalizados como medidores de batería.
    \end{itemize}
    \item \textbf{Configuración y estilos} (\texttt{.streamlit/config.toml}):
    \begin{itemize}
        \item Personalización del tema (colores institucionales, tipografía) y disposición de la página.
    \end{itemize}
\end{enumerate}

\section{Modelo matemático (resumen)}

El modelo se basa en los siguientes pasos:

\begin{itemize}
    \item \textbf{Carga doméstica total}: suma de horas semanales en limpieza, cocina, lavado y cuidados directos.
    \item \textbf{Fatiga cognitiva}: función sigmoidea que transforma la carga en un índice entre 0 y 1, con parámetros que dependen del género (mayor pendiente para mujeres) y estrato (mayor factor para estratos bajos).
    \item \textbf{Horas de estudio efectivas}: horas de estudio declaradas multiplicadas por \((1 - \text{fatiga})\).
    \item \textbf{Estimación del promedio (PPA)}: regresión lineal con coeficientes calibrados, que incluye horas efectivas, estrato, género, número de dependientes e interacción género-fatiga.
    \item \textbf{Shadow-Score}: normalización de la pérdida de PPA respecto a un máximo teórico, combinada con la carga total.
\end{itemize}

Este modelo es transparente y ajustable a medida que se disponga de más datos.

\section{Flujo de usuario (workflow)}

El usuario interactúa con la aplicación de la siguiente manera:

\begin{enumerate}
    \item \textbf{Página de inicio}: presentación del proyecto y su propósito.
    \item \textbf{Formulario de entrada} (barra lateral o panel izquierdo):
    \begin{itemize}
        \item Selección de género.
        \item Deslizador para estrato (1-6).
        \item Número de dependientes (hijos, adultos mayores, etc.).
        \item Horas semanales en cada tarea doméstica (campos numéricos con paso de 0.5).
        \item Horas de estudio semanales.
        \item Promedio actual (opcional).
    \end{itemize}
    \item \textbf{Cálculo}: al presionar ``Calcular mi Shadow-Score'', se ejecutan las funciones del modelo y se muestran los resultados.
    \item \textbf{Visualización de resultados}:
    \begin{itemize}
        \item Shadow-Score con barra de progreso y etiqueta cualitativa.
        \item Fatiga cognitiva representada como un medidor circular (batería).
        \item Horas de estudio efectivas comparadas con las declaradas.
        \item Promedio estimado (y diferencia con el actual, si se ingresó).
        \item Gráfico de radar con las horas dedicadas a cada tarea.
    \end{itemize}
    \item \textbf{Simulación de corresponsabilidad}:
    \begin{itemize}
        \item Botón ``Simular corresponsabilidad'' que reduce las horas domésticas en un porcentaje definido (ej. 40\%, basado en metas ODS).
        \item Se muestran dos columnas: \textbf{Escenario actual} vs. \textbf{Escenario con corresponsabilidad}, con métricas comparativas y flechas indicando mejora.
    \end{itemize}
    \item \textbf{Información adicional}: un expander con la metodología y fuentes.
\end{enumerate}

\section{Consideraciones de calidad y buenas prácticas}

\begin{itemize}
    \item \textbf{Código limpio}: seguir PEP 8, funciones bien documentadas, typing.
    \item \textbf{Pruebas}: al menos pruebas unitarias para las funciones críticas (ej. \texttt{calcular\_fatiga}, \texttt{estimar\_ppa}).
    \item \textbf{Manejo de errores}: capturar entradas inválidas y mostrar mensajes amigables.
    \item \textbf{Accesibilidad}: usar etiquetas claras, contraste suficiente, y probar con lectores de pantalla.
    \item \textbf{Internacionalización}: aunque inicialmente en español, estructurar el código para permitir cambio de idioma mediante diccionarios.
    \item \textbf{Seguridad}: no almacenar datos personales sin consentimiento; si se implementa base de datos, anonimizar.
\end{itemize}

\section{Posibles ampliaciones futuras}

\begin{itemize}
    \item \textbf{API REST}: separar la lógica de negocio en una API con FastAPI, permitiendo que otros servicios (chatbots, aplicaciones móviles) consuman los mismos cálculos.
    \item \textbf{Modelos de machine learning}: cuando se disponga de suficientes datos, entrenar un modelo predictivo más complejo y comparar con el modelo paramétrico.
    \item \textbf{Recomendaciones personalizadas}: sugerir recursos de apoyo (becas, grupos de cuidado compartido, etc.) según el perfil.
    \item \textbf{Dashboard de investigación}: visualizar estadísticas agregadas de los usuarios (con su consentimiento) para alimentar informes académicos.
\end{itemize}

\section{Conclusión}

El proyecto \textbf{Shadow-Score Académico} es una iniciativa con alto potencial de impacto social y académico. La elección de Streamlit como plataforma de desarrollo permite construir una herramienta robusta, interactiva y fácil de mantener en un plazo razonable, incluso con recursos limitados. El plan aquí propuesto garantiza un avance estructurado, con entregables claros y la flexibilidad para incorporar mejoras progresivas.

La combinación de un modelo matemático sólido, una interfaz amigable y la capacidad de simular escenarios de corresponsabilidad ofrece una experiencia reveladora para los usuarios, contribuyendo a la concienciación sobre la desigualdad de género en el ámbito universitario y proporcionando evidencia para la formulación de políticas institucionales.
